# SMTP Relay - Docker

A production-ready Docker container providing an authenticated SMTP submission server with **Postfix**, **Cyrus SASL (PAM)** authentication, **OpenDKIM** signing, and **TLS** encryption. Designed as a lightweight, secure relay for outbound mail with optional upstream provider support.

## Features

- **Postfix SMTP Server** - Full-featured MTA running in submission mode on port 587
- **Cyrus SASL + PAM Authentication** - Local system accounts with `/usr/sbin/nologin` shell (users cannot log in interactively or via SSH)
- **OpenDKIM Signing** - Automatic DKIM email signing via milter for improved deliverability
- **TLS Encryption** - STARTTLS required for authentication, TLSv1.2/TLSv1.3 only
- **SMTP Relay Support** - Optional relay through upstream providers (Gmail, Amazon SES, SendGrid, Mailgun, SMTP2GO, Office365, or custom)
- **Interactive Installer** - Color-coded, step-by-step setup wizard
- **Persistent Storage** - DKIM keys, TLS certs, and user password hashes survive container rebuilds
- **Health Checks** - Built-in Docker healthcheck for Postfix, saslauthd, and OpenDKIM
- **Rate Limiting** - Per-client connection, message, and auth rate limits
- **Security First** - No open relay, strong ciphers, disabled VRFY, nologin users
- **RFC 5321 / RFC 5322 Compliant** - Modern Postfix 3.6+ compatibility level

## Prerequisites

Before running the installer, ensure your host has:

- **Docker** 20.10+ and **Docker Compose** v2
- **OpenSSL** (for the installer to generate certs and DKIM keys)
- **Root or sudo access** (the installer writes to `configs/`, `data/`, and `docker-compose.yml`)
- **Port 587/tcp** open in your firewall (mapped from the container to the host)
- **A registered domain** with access to its DNS records

Verify your environment:

```bash
docker --version
docker compose version
openssl version
```

## Quick Start

### 1. Clone and run the installer

```bash
git clone https://github.com/kennarionwong/smtp-relay.git
cd smtp-relay
sudo bash install.sh
```

The installer is interactive and will prompt you for:

1. **Domain and hostname** (e.g. `example.com` / `mail.example.com`)
2. **SMTP authentication** - add one or more users (interactive or skipped)
3. **Relay provider** - direct delivery or upstream relay
4. **DKIM** - selector, key size (default 2048)
5. **TLS certificate source** - self-signed, existing files, or Let's Encrypt

### 2. Add the DNS record

The installer prints the exact DKIM TXT record. Add it (and A/MX/SPF records - see [DNS Records](#dns-records)) to your DNS provider before testing deliverability.

### 3. Deploy with Docker Compose

```bash
docker compose up -d --build
```

### 4. Verify

```bash
# Check container status
docker compose ps

# Watch startup logs
docker compose logs -f

# Verify healthcheck passes
docker inspect --format='{{.State.Health.Status}}' smtp-relay

# Test STARTTLS handshake
openssl s_client -starttls smtp -connect localhost:587
```

## Configuration

### Domain and Hostname

Set your mail domain and hostname during install. These are baked into:

- `main.cf` - `myhostname` and `mydomain`
- `docker-compose.yml` - container `hostname:` and `domainname:`
- DKIM signing table and trusted hosts
- Self-signed certificate SAN entries

```
Domain: example.com
Hostname: mail.example.com
```

### SMTP Authentication (Cyrus SASL + PAM)

SMTP users are local Linux system accounts with `/usr/sbin/nologin` shell. They:

- **Can** authenticate against the SMTP submission port
- **Cannot** log in interactively or via SSH
- Are members of the `smtpusers` group
- Are recreated as system accounts at every container start from `data/users/smtp-users`

Authentication flow: Postfix → `saslauthd` (PAM mechanism) → `pam_unix.so` → system accounts.

Password hashes are stored as SHA-512 (`openssl passwd -6`) in `data/users/smtp-users` and re-applied via `chpasswd -e` on every container start.

#### Managing users

```bash
# Add user (prompts for password)
docker exec -it smtp-relay setup-auth.sh add username

# Add user with password (non-interactive)
docker exec -it smtp-relay setup-auth.sh add username 'S3cur3P@ss!'

# List users
docker exec -it smtp-relay setup-auth.sh list

# Change password
docker exec -it smtp-relay setup-auth.sh passwd username

# Verify credentials work (uses testsaslauthd)
docker exec -it smtp-relay setup-auth.sh verify username password

# Remove user
docker exec -it smtp-relay setup-auth.sh remove username
```

### TLS Certificates

Three options, configured in the installer:

1. **Self-signed** (default) - Generates a 4096-bit CA, 2048-bit server key, and a 10-year certificate with proper SAN entries (your domain, `*.domain`, `localhost`, `127.0.0.1`).
2. **Existing certificates** - Provide paths to your own `.crt` and `.key` files; the installer copies them into `data/certs/`.
3. **Let's Encrypt** - Mount `/etc/letsencrypt` from the host. The installer expects `<domain>/fullchain.pem` and `<domain>/privkey.pem`.

If the container is started without certificates present, the entrypoint auto-generates a temporary self-signed cert so the service can still start.

### Relay Configuration

If you choose to relay through an upstream provider during install, the following providers are preconfigured:

| Provider      | Host                              | Port |
|---------------|-----------------------------------|------|
| Gmail         | `smtp.gmail.com`                  | 587  |
| Amazon SES    | `email-smtp.<region>.amazonaws.com` | 587 |
| SendGrid      | `smtp.sendgrid.net`               | 587  |
| Mailgun       | `smtp.mailgun.org`                | 587  |
| SMTP2GO       | `smtp.smtp2go.com`                | 587  |
| Office365     | `smtp.office365.com`              | 587  |
| Custom        | (prompted)                        | (prompted) |

Credentials are written to `configs/postfix/sasl_passwd` and the relay host is configured in `main.cf`. The `sasl_passwd.db` hash map is generated at container start.

### DKIM Setup

The installer generates DKIM keys (default 2048-bit RSA) using `opendkim-genkey` (or an `openssl` fallback) and displays the exact DNS TXT record to publish:

```
Name:  mail._domainkey.example.com
Type:  TXT
Value: "v=DKIM1; k=rsa; p=MIIBIjANBgkqhkiG9w0BAQEFAAOCAQ8AMIIBCgKCAQEA..."
```

DKIM signing is applied via the OpenDKIM milter (socket: `local:/opendkim/opendkim.sock`, protocol 6).

You can also generate DKIM keys independently using the helper script:

```bash
docker exec smtp-relay generate-dkim.sh example.com mail 2048
```

## DNS Records

Before sending mail, publish these records at your DNS provider:

```
# A Record - point your mail hostname at the server
mail.example.com.    IN    A    <your-server-ip>

# MX Record - tell other MTAs where to deliver mail for your domain
example.com.         IN    MX   10 mail.example.com.

# SPF Record - authorize this server to send for your domain
example.com.         IN    TXT  "v=spf1 mx a ~all"

# DKIM Record - from the installer output
mail._domainkey.example.com.  IN  TXT  "v=DKIM1; k=rsa; p=..."

# DMARC Record (strongly recommended)
_dmarc.example.com.  IN    TXT  "v=DMARC1; p=none; rua=mailto:dmarc@example.com"
```

Verify propagation:

```bash
dig A mail.example.com
dig MX example.com
dig TXT mail._domainkey.example.com
```

## Testing

### Test STARTTLS handshake

```bash
openssl s_client -starttls smtp -connect localhost:587
```

### Test authenticated send with swaks

```bash
swaks --to recipient@example.com \
      --from sender@example.com \
      --server localhost:587 \
      --auth LOGIN \
      --auth-user username \
      --auth-password 'yourpassword' \
      --tls
```

### Manual SMTP session

```bash
openssl s_client -starttls smtp -connect localhost:587
# Then inside the SMTP session:
EHLO test
AUTH LOGIN
# (paste base64-encoded username)
# (paste base64-encoded password)
MAIL FROM:<sender@example.com>
RCPT TO:<recipient@example.com>
DATA
Subject: Test
Test email body
.
QUIT
```

Generate the base64 credentials quickly:

```bash
echo -n 'username' | base64
echo -n 'yourpassword' | base64
```

### Verify DKIM signature on outgoing mail

After sending a test message, view the raw source in your recipient client and confirm a `DKIM-Signature:` header is present for your selector.

## Security

### No open relay

`smtpd_recipient_restrictions` only accepts mail from SASL-authenticated clients or the loopback interface:

```
smtpd_recipient_restrictions =
    permit_sasl_authenticated,
    permit_mynetworks,
    reject_unauth_destination
```

### TLS hardening

- `smtpd_tls_auth_only = yes` - authentication requires STARTTLS
- `smtpd_tls_security_level = encrypt` on the submission service
- Only TLSv1.2 and TLSv1.3 enabled (SSLv2, SSLv3, TLSv1.0, TLSv1.1 disabled)
- Strong mandatory ciphers

### Rate limiting

- 30 client connections per minute
- 60 messages per minute per client
- 10 AUTH attempts per minute per client
- Submission service: 20 concurrent client connections max

### Additional hardening

- `disable_vrfy_command = yes`
- `smtpd_helo_required = yes`
- Sender restrictions reject non-FQDN and unknown sender domains
- SMTP users have `/usr/sbin/nologin` shell and `smtpusers` group membership
- SASL authentication requires TLS (`smtpd_sasl_security_options = noanonymous, noplaintext`)
- `permit_sasl_authenticated` is the only way to relay externally

### File permissions

| Path                        | Mode | Owner          |
|-----------------------------|------|----------------|
| `data/dkim/*/*.private`     | 600  | `opendkim`     |
| `data/certs/mail.key`       | 600  | `root`         |
| `data/certs/mail.crt`       | 644  | `root`         |
| `data/users/smtp-users`     | 600  | `root`         |
| `/etc/opendkim/keys/`       | 700  | `opendkim`     |

## Troubleshooting

### Authentication fails

```bash
# Is saslauthd running?
docker exec smtp-relay pgrep -a saslauthd

# Test PAM auth directly
docker exec smtp-relay testsaslauthd -u username -p password -f /var/run/saslauthd/mux -r smtp

# Check authentication log
docker exec smtp-relay tail -50 /var/log/auth.log

# Inspect SASL config
docker exec smtp-relay cat /etc/postfix/sasl/smtpd.conf
```

### DKIM not signing mail

```bash
# Are keys present?
ls -la data/dkim/example.com/

# Is OpenDKIM running?
docker exec smtp-relay pgrep -a opendkim

# Check OpenDKIM socket
docker exec smtp-relay ls -la /var/spool/postfix/var/run/opendkim/

# Inspect mail log for milter errors
docker exec smtp-relay grep -i 'opendkim\|milter' /var/log/mail.log

# Verify DNS is published
dig TXT mail._domainkey.example.com
```

### TLS handshake fails

```bash
# Check certificate files
docker exec smtp-relay ls -la /data/certs/

# Inspect certificate
docker exec smtp-relay openssl x509 -in /data/certs/mail.crt -noout -subject -dates

# View full STARTTLS debug
openssl s_client -starttls smtp -connect localhost:587 -debug
```

### Relay not delivering

```bash
# Verify relay host and credentials
docker exec smtp-relay postconf relayhost
docker exec smtp-relay cat /etc/postfix/sasl_passwd
docker exec smtp-relay postmap -q '[relayhost]:587' hash:/etc/postfix/sasl_passwd

# Inspect the queue
docker exec smtp-relay postqueue -p
docker exec smtp-relay postcat -q <queue_id>
```

### Container won't start

```bash
docker compose logs
# Validate Postfix config
docker run --rm -v $PWD/configs/postfix:/etc/postfix smtp-relay:latest postfix check
```

## Reconfiguration

The installer is **idempotent** - re-running it safely updates existing configuration files in place.

```bash
sudo bash install.sh        # Re-run with new answers
docker compose down
docker compose up -d --build
```

The installer regenerates `configs/postfix/main.cf`, `master.cf`, `sasl/smtpd.conf`, `configs/opendkim/*`, `configs/pam/smtp`, and `docker-compose.yml`. It preserves the contents of `data/` (your keys, certs, and users).

To reconfigure TLS or DKIM **without** losing users, re-run the installer and choose the same values; only the targeted subsystem will be regenerated.

## Backups and Upgrades

### What to back up

The `data/` directory contains all persistent state. Back it up regularly:

```
data/
├── dkim/           # DKIM private/public keys (critical - if lost, you must regenerate and update DNS)
├── certs/          # TLS certificates and keys
└── users/
    └── smtp-users  # User password hashes (SHA-512 encrypted)
```

```bash
# Create a backup
tar -czf smtp-relay-backup-$(date +%Y%m%d).tar.gz data/

# Restore from backup
tar -xzf smtp-relay-backup-20260101.tar.gz -C /path/to/smtp-relay/
```

You should also back up the `configs/` directory if you have made manual customizations outside of the installer.

### Upgrading

```bash
# Pull latest changes
git pull origin main

# Rebuild the container
docker compose down
docker compose up -d --build

# Verify
docker compose logs -f
```

The container is designed so that a rebuild picks up existing `data/` volumes automatically - DKIM keys, TLS certs, and user accounts persist across rebuilds.

### Container image updates

```bash
# Pull latest base image and rebuild
docker compose build --no-cache
docker compose up -d
```

### Key files and their roles

| File | Purpose |
|------|---------|
| `entrypoint.sh` | Container startup: validates configs, starts services (rsyslog → saslauthd → OpenDKIM → Postfix), creates users from persistent storage, tail-follows logs |
| `install.sh` | Host-side setup wizard: collects domain/auth/relay/DKIM/TLS config, generates all `configs/` files, generates certs & DKIM keys, creates `docker-compose.yml` |
| `configs/postfix/main.cf` | Postfix global configuration: hostname, TLS, SASL, DKIM milter, rate limits, security restrictions |
| `configs/postfix/master.cf` | Postfix service definitions: submission service on port 587 with forced TLS+SASL+milter |
| `configs/postfix/sasl/smtpd.conf` | Cyrus SASL: delegates auth to saslauthd via PAM, supports LOGIN and PLAIN mechanisms |
| `configs/pam/smtp` | PAM policy: authenticates against local system accounts via `pam_unix.so` |
| `configs/opendkim/opendkim.conf` | OpenDKIM: runs as signer+verifier (sv mode), references KeyTable/SigningTable/TrustedHosts |
| `configs/opendkim/KeyTable` | Maps DKIM selector to domain:selector:keyfile |
| `configs/opendkim/SigningTable` | Maps domains to DKIM selectors |
| `configs/opendkim/TrustedHosts` | List of hosts whose mail will be signed (loopback + your domain/hostname) |
| `scripts/healthcheck.sh` | Docker healthcheck: verifies Postfix (master), saslauthd, OpenDKIM processes are running, TLS certs exist, and port 587 is listening |
| `scripts/setup-auth.sh` | In-container user manager: add/remove/list/passwd/verify SMTP users |
| `scripts/generate-dkim.sh` | Standalone DKIM key generator with DNS record output |
| `scripts/generate-tls.sh` | Standalone TLS certificate generator (self-signed, existing, or Let's Encrypt) |
